Modify default parameters

____________

 

This section will guide you to modify JSIM default parameters. The main window let you change parameters of four sections:

 

  • Class Parameters

  • Station Parameters

  • Simulation Parameters

  • Finite Capacity Region (FCR) Parameters

 

Class Parameters

 

These settings let you chose the default class parameters to be used whenever a new class is created. For a more detailed view on classes see here.

 

Name refers to the default name of a class that will be followed by a progressive number.

 

Type is the default type of the class (closed or open).

 

Priority is a number used to define the priority order of the classes.

 

Population (only for closed classes) is the default total number of users of closes classes.

 

Distribution (only for open classes) is the default distribution type of open classes.

 

Station Parameters

 

These settings are general parameters for every type of station, for a more detailed description of station types see here

 

Name is the default name of a new station that will be followed by a progressive number.

 

Type is the default type of a new station. (Possible choices are Server, Fork, Join, LoadSplitter, Delay)

 

Queue Capacity (for Server, Fork station types) represents the default total amount of users that can reside in the station and could be infinite, if infinite checkbox is checked.

 

Number of Servers (for Server station type) represents the default number of servers associated to a Server station.

 

Queue Strategy (for Server and Fork station types) is the default queue strategy used in a station. (Choices are FCFS and LCFS with or without priority)

 

Service Strategy (for Server station type) is the default service time distribution used for a service.

 

Delay Service Strategy (for Delay station type) is the default service time distribution used in a Delay station for every class.

 

Routing Strategy (for Delay, Server, Join, LoadSplitter types) is the default routing strategy used in a station for every class.

 

Fork Blocking (for Fork station type) is the default maximum number of users allowed before blocking occurs in a Fork Station. This could be infinite is appropriate checkbox is checked.

 

Fork Tasks per Link (for Fork station type) represents the number of tasks for every outgoing link of a Fork station.

 

Simulation Parameters

 

These parameters defines the way the simulation behaviours. For another view on them see here (for simulation) or here (for Confidence Interval and Max Relative Error)

 

Measure Confidence Interval is the percentage of the confidence interval.

 

Measure Max Relative error is the value of the maximum relative error allowed.

 

Simulation Seed represents the starting seed of a simulation.

 

Maximum Duration defines the maximum time a simulation could take before stopping (in seconds).

 

Maximum Number of Samples is the maximum number of sample used in a simulation.

 

Polling Interval is the time used between every two samples in the simulation.

 

N° of represented class defines the number of class that will be represented in the simulation graph. It's possible to select or unselect the animation with the near checkbox.

 

Finite Capacity Region (FCR) Parameters

 

These settings affect the default value used when you create a new Finite Capacity Region. For a more detailed view on Finite Capacity Regions see here

 

Name represents as always the default name used for a new Finite Capacity Region that will be followed by a number.

 

Global Region Capacity is the total number of users that could reside in the global region. This could be infinite or not.

 

Region Capacity per Class is the partial number of users for a Region per class.

 

Drop tells if Finite Capacity Region have to drop or not packets by default.
