Define What-If Analysis

_________________

 

In this section it will be told how to set and how to use a What-If Analysis.

 

A What-If Analysis consists on a sequence of simulations in which one or more parameters are modified from a starting value to an ending one.

In this way you can compare different results depending on different parameters value, so what-if analysis simulation view is fairly different from standard JSIM simulation view.

Activating the What-If Analysis

Selecting the What-If Analysis Parameter

 

Activating the What-If Analysis

After have created a new model, from the Tab list just select the "What-If Analysis" tab.

 

 

As what-if analysis is disabled by default, you have to click in the top right section of the tab the "Enable what-if analysis" checkbox to activate it.

 

 

Selecting the What-If Analysis Parameter

Now it's time to select which parameter you want to control in the proper listbox:

 

 

Whenever you select a what-if analysis parameter the bottom section of the window will change in association with that parameter. A short description on selected parameter will appear on the right bottom section of the current window to help in modifying values.

Selectable parameters are different depending on the number and on the type of classes in current model:

  • If you have single classes you can select the parameters you want to change during simulation
  • If you have multiple classes you can select the parameters and chose if you want to apply them to all classes or just to one class.

 

Number of Customers (only if there is a model with closed classes)

 

JSIM repeats the simulation changing the number of jobs in each interaction, starting from the number inserted when you have defined the closed classes to the number inserted here, in the form "To".

 

In this figure (example of what-if analysis on only one class) you can see the initial value that is the sum of the populations of closed classes (modifiable from Define menu in "Customer Classes".

You must insert the final value "To", the steps for the simulation and the relative closed class. The new simulation will be calculated on the selected class only.

If you have chosen "all-classes", the proportion of jobs in different classes will be kept constant, but the number of steps that can be executed may be very small.

The table shows the number of customers in each class, and the proportion of jobs for each class respect the sum of all jobs. The sum of all proportions is always 1.

 

Arrival Rate: (only if there are open classes)

 

Arrival rate is the number of job arriving in a station/device during a period of time T

 

Select the class or all classes to repeat the simulation with different arrival rates. If you select all open classes, the measure is counted in % otherwise in jobs per second (j/s) .

 

This figure shows an example of the arrival rate parameter with all classes selected.

  1. Insert the value "To" that represents the final arrival rate (in this case in percentage).
  2. Insert the number of steps that JSIM will perform to reach value "To", starting from the initial value you have selected when you have defined the class. Initial value depends on the type of distribution selected in Arrival Rate for that class.

 

Service time (for all types of classes)

 

Service Time is the time required per visit in a station k.

 

This is the figure you see, for example, when you select one class only. The simulation will be repeated changing the Service Time for the station you select in this form. The simulation will be repeated n times according to the number of steps you have chosen. The initial value is calculated on the type of class, the arrival distribution and the number of jobs in the initial state.

You must insert the final value for the Service Time, the station and the Class you want to apply to the simulation.

Select one class or All, if you choose a class only the measures is in Seconds (s), otherwise (All Classes-multiple classes model) is in %. The simulation engine repeats the calculations from initial value to final value in n step and will show you the results graph.

 

Seed (for all types of classes)

 

 

The Simulation engine uses a random algorithm to generate numbers and for each calculation it uses a "seed". However these numbers are not really random because the algorithm uses the usual seed to generate it. If you change the seed number you can obtain various random numbers that are different from other numbers which were previously generated.

If you repeat a simulation with usual seed, you obtain usual numbers and this doesn't realize a random algorithm or random numbers.

To generate different random numbers you should change the seed each time the simulation starts.

The initial value is already inserted in "Simulation Parameters" in Define menu, and in this type of simulation you should insert only the number of steps.
