Define simulation parameters

_________________

 

By clicking Simulation tab you can set up the Simulation Parameters and Initial State for your model.

 

 

Simulation parameters:

In the top section of this windows you can edit general simulation parameters.

 

  1. Simulation Seed: The seed is a number used by random engine to generate casual numbers. As this numbers are pseudo-casual if you change the seed, you will obtain random numbers that are different from others. If you use the same seed number, the random engine will produce the same numbers and the simulation results will be equal.

    Insert a number if you want a personalized simulation, or check "random" if you don't want to define a particular seed.

     

  2. Maximum duration (sec): represents the time in seconds that the simulation will run at last. If simulation ends before this amount of seconds, it won't be affected by this parameter. If you don't want to specify a maximum time you can check "Infinite" in check-box. The "infinite" setting forces the simulation to stop only when it has reached a solution but it can take a long computation time.

     

  3. Maximum number of samples: it's the greatest number of samples for one measure that JSIM collects before ending the simulation. During a simulation, measurement can be stopped by:

    4.  

    - Success, if the results have reached the required Confidence Interval and the Max Relative Error

    - Failure, if the simulation has analyzed the maximum number of samples

    - Failure, if timeout happens before succeeding.

     

    In "Max number of samples" insert the number of samples that are used by the simulation engine.

     

  4. Representation Interval (sec): you can insert a number of seconds that are represented in the graphics plotted by the simulation engine in the results. As always, an higher value will take more time to show changes in the simulation while a lower value will give an "immediate" look of the model behaviour.

Initial state:

Initial state is the number of jobs that are in the station queue before starting a simulation. It shows the network situation at time 0.

For closed customer classes, all jobs are initially allocated in their reference station. You can at any time modify this allocation considering that the total number of jobs should be the one defined in the classes page of tabs menu (link). In open customer classes it's possible to initialize every station with any desired number of jobs.
