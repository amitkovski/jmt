Error and Warning Messages

_________________

 

When you decide to start the simulation, JSIM analyses all the network you have built.

 

If JSIM encounters any errors, it will show you this windows with error type and description:

 

 

In the following table you can see some common problems with their solution:

 

If you have followed the correct steps to define your model, this error does not appear. For each class you must define a Reference Station that is used in the analysis. To define a reference station read here (link) or press "customer classes" from Define menu and insert it.

 

When you have inserted a station, you must link it with another device in the model. To connect two elements in the model see "Step3: connections" (link). Only the Sink station doesn't have a forward link.

 

Each sink station can be backward linked, otherwise all the jobs of open classes that have been processed can't leave the network as the model requires. See how to connect two elements here (link). Only the Source station doesn't have a backward link.

This error appears when you try to start the simulation but no performance index is defined. To correct it, select at least one index in "performance Indices" from Define menu. See also "How to define performance indices" (link)

 

Fork and Join stations must be inserted together in the network . If you have inserted only one of the two stations, JSIM will show an error or warning message. To debug it, insert the associated station in the model and rebuild the topology. See "Step3: connections" (link) and insert the missing device

 

This error appears when you have forgotten to define a model input. The customer classes are important parameters for the simulation, You can't do anything without them. See "Step 1: Classes" (link) or "How to define classes" (link)

 

When you have defined the model output, you have inserted some equal indices. You must insert only one index with same parameters. Select "Performance Indices " and delete the equal ones. See Also: "Step 1: Performance indices " (link) or "How to define performance indices " (link)

 

When you use a closed class there is a fixed number of customers that circulate in the network. A customer therefore can't leave it. If you linked the network only with the sink this error appears. To correct it, feedback connect the element to itself or to others devices. To connect two elements in the model see "Step3: connections" (link)

 

When you have built your model with JSIM and you want to import it to JMVA, you can see this warning because JMVA doesn't implement Round Robin strategy. If you want you can correct it, otherwise press "continue" and the strategy will change from "Round Robin" to "Random".

 

This message appears when you try to import the model to JMVA. If you have defined different Service Time for stations (ex: Station1 = exp(1), Station2 = exp(2)) JSIM will show this warning. If you press "continue" the Service Times in JMVA are resetted to the default value, otherwise you can modify Service Times before importing the model.

 

When you insert a new performance index, you must select the station on which it is defined. If no station is selected, this error appears. To correct it, click on Performance Index from Define menu and select the station in the correct index. (link)
